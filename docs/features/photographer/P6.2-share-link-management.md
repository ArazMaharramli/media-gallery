# P6.2: Share Link Management

**Description:** View, manage, and revoke share links for an event.

**Entry Point:** Dashboard - new "Share Links" panel (similar to Upload Links)

---

## Acceptance Criteria

- [ ] "Share Links" button in dashboard header (next to "Upload Links")
- [ ] Collapsible panel shows all view tokens for the event
- [ ] Each link displays: token preview, creation date, access count
- [ ] Copy button for each existing link
- [ ] Revoke/deactivate button for each link
- [ ] Confirmation prompt before revoking
- [ ] Revoked links show as inactive (grayed out)
- [ ] Revoked links return 404 when accessed

## Data Model Changes

Update `ViewToken` model in Prisma:

```prisma
model ViewToken {
  id        String   @id @default(cuid())
  eventId   String
  token     String   @unique
  active    Boolean  @default(true)    // NEW
  accessCount Int    @default(0)       // NEW (optional)
  createdAt DateTime @default(now())
  event     Event    @relation(fields: [eventId], references: [id], onDelete: Cascade)
}
```

---

## API

### Get all view tokens for event
`GET /api/events/:id/view-tokens`

Response:
```json
{
  "success": true,
  "data": [
    {
      "id": "clx...",
      "token": "abc123def456",
      "active": true,
      "accessCount": 12,
      "createdAt": "2024-03-23T10:00:00Z"
    }
  ]
}
```

### Revoke a view token
`PATCH /api/events/:id/view-tokens/:tokenId/revoke`

Response:
```json
{
  "success": true,
  "data": { "revoked": true }
}
```

---

## Gallery Access Changes

Update `/api/gallery/:token` to check `active` status:
- If `active === false`, return 404 "Gallery not found"
- If `active === true`, serve gallery and increment `accessCount`
